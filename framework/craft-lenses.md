# Craft lenses — a diagnostic toolkit

*Diagnostic, **not** generative — for the *interrogation* pass, not the *dreaming* pass. Dream the
story, then run the lenses to find blind spots; never force the story to fit a grid (procrustean
failure). A good lens asks a yes/no/deliberate-choice question; when the answer surprises you, it
found something. Use a few, internalize them, discard the rest. Point the lenses **hardest** where
the author over-identifies (often the protagonist) — that's where judgment is most compromised.*

## The toolkit
**Motivation**
1. **Want vs. Need + The Lie** — conscious external *want*; internal *need*; the **Lie** they
   believe is the engine between. (Most load-bearing character lens.)
2. **Inner contradiction + opposition web** — a live character wants two incompatible things; every
   major character embodies a *distinct* stance on the theme (cast = argument).

**Action (scene)**
3. **Does the scene turn a charged value?** (McKee) +→– or –→+. If nothing changes, it's
   exposition, not a scene.
4. **Causality: "therefore / but," not "and then."** Beats connect by cause, not sequence.
5. **Scene → Sequel rhythm** (Swain) — action (goal/conflict/setback) then reaction/dilemma/
   decision. Controls micro-pacing.

**Plot / Pacing**
6. **MICE** (Card) — which thread *frames* the book: Milieu / Inquiry / Character / Event? Open a
   thread first → close it last; nested threads close inside-out.
7. **Try/fail escalation (yes-but / no-and)** for the middle + a **reader-knowledge map** (who
   knows what, when) for dramatic-irony control. Don't flatline — oscillate hope/dread.

## Live audit (per project)
*Run the lenses; log the gaps they surface — and their resolutions — in the project's own
`craft-audit.md`, so this generic catalog stays clean and shareable.*
